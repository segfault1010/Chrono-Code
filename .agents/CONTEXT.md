# Chronocode — Project Context & Current State

## 🚀 Overview
**Chronocode** is a "Git History Intelligence" tool. It allows users to paste any public GitHub repository URL into a search bar, instantly parses the repository's commit history, and presents it in a beautiful timeline. Users can then click "AI Explain" on any commit to get a streamed, architectural explanation of the code changes, generated by Gemini.

## 🛠 Tech Stack
- **Frontend**: Next.js (React), Tailwind CSS v4, Vanilla CSS (for glowing mesh gradients and custom animations).
- **Backend**: Express.js (Node.js API), `simple-git` for repository manipulation.
- **Database**: Supabase (PostgreSQL) storing repositories, commits, and cached AI explanations.
- **AI**: Google Gemini (via `@google/genai`) for streaming commit explanations.

## ✅ What We've Built So Far
We have successfully built a fully functioning, highly optimized MVP.

### 1. UI & Aesthetics
- Completely redesigned the UI to feature a premium, deep-space black aesthetic with neon accents.
- Implemented a floating, glassmorphism search bar on the homepage.
- Added a radiant "glowing mesh" background gradient to the bottom of the page that doesn't conflict with layout spacing.
- The repository timeline displays the exact date and time for every commit in a sleek, vertically connected layout.

### 2. High-Performance Indexing Engine
- Repositories now load **instantaneously**, even massive ones like `facebook/react` or `microsoft/vscode` (takes ~2 to 6 seconds).
- **How it works:** The backend uses a heavily optimized Git clone command:
  `git clone --bare --single-branch --filter=blob:none --depth=100`
- The system skips counting line insertions/deletions during the initial indexing phase, which completely bypasses the need to fetch file blobs from GitHub, resulting in zero blocking network calls during `git log`.
- Background rate limits have been increased to prevent the frontend polling loop from triggering "Network Errors" (429 Too Many Requests) during development.

### 3. Streaming AI Explanations
- The "AI Explain" feature fetches the specific diff for a commit and streams the explanation character-by-character back to the client using **Server-Sent Events (SSE)**.
- Users can click "AI Explain" on **multiple commits simultaneously**; the state is managed via a dictionary/map in React so no explanations are closed when another is opened.

### 4. Zero-Friction Onboarding
- If the user leaves the input blank, it defaults to a popular repository.
- Added a "Demo Repositories" section with popular, fast-loading projects (`axios/axios`, `reduxjs/redux`, `npm/node-semver`, etc.) that users can click to bypass manual entry.
- Auto-formatter on the frontend allows users to just type `owner/repo` instead of the full HTTPS URL.
